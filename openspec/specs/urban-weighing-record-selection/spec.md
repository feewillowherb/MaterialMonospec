## Purpose

Defines the click-based row selection interaction for vehicle record lists in the UrbanAttendedWeighingWindow, including visual feedback for the selected row and MVVM command binding patterns.

## Requirements

### Requirement: Vehicle record row selection via Button Command

The vehicle record list in UrbanAttendedWeighingWindow SHALL use Button elements with Command binding for row selection. Each row SHALL be a Button styled with `transparent-button` class, binding its `Command` to `SelectListItemCommand` on the parent ViewModel via `$parent[ItemsControl]` syntax, and passing the current item as `CommandParameter`.

#### Scenario: User clicks on a vehicle record row
- **WHEN** user clicks anywhere on a vehicle record row
- **THEN** the `SelectListItemCommand` SHALL execute with the clicked item as parameter
- **AND** `SelectedListItem` on the ViewModel SHALL be set to the clicked item

#### Scenario: Click does not interfere with row child elements
- **WHEN** user clicks on any area of the row (including text columns, status badges, or action text)
- **THEN** the selection SHALL still trigger reliably without being intercepted by child elements

### Requirement: Selected row visual highlighting

The currently selected vehicle record row SHALL display a distinct background color (`#E3F2FD`) using `EqualityToColorConverter` with `MultiBinding`. Unselected rows SHALL have a transparent background.

#### Scenario: Row becomes selected
- **WHEN** a vehicle record row is selected (item equals SelectedListItem)
- **THEN** the row background SHALL change to `#E3F2FD`

#### Scenario: Previously selected row becomes deselected
- **WHEN** a different row is selected
- **THEN** the previously selected row background SHALL revert to transparent
- **AND** the newly selected row background SHALL change to `#E3F2FD`

### Requirement: SelectListItem generates ReactiveCommand

The `SelectListItem` method on `UrbanAttendedWeighingViewModel` SHALL have the `[ReactiveCommand]` attribute, which SHALL generate a `SelectListItemCommand` property available for XAML binding.

#### Scenario: Command property is generated
- **WHEN** the ViewModel is compiled
- **THEN** a `SelectListItemCommand` property SHALL be generated by the ReactiveUI source generator
- **AND** the command SHALL accept `UrbanWeighingListItemDto?` as parameter

### Requirement: Approval column uses TextBlock instead of nested Button

The "审批" (approval) column in the vehicle record list SHALL use a `TextBlock` element instead of a `Button` to avoid nested Button event conflicts.

#### Scenario: Approval column displays as text
- **WHEN** the vehicle record list renders
- **THEN** the approval column SHALL display "审批" as a TextBlock
- **AND** clicking on it SHALL trigger row selection (via parent Button) rather than being intercepted

### Requirement: No code-behind event handler for row selection

The `UrbanAttendedWeighingWindow.axaml.cs` code-behind SHALL NOT contain any `OnRecordClick` or `PointerPressed` event handler for row selection. All selection logic SHALL be handled through MVVM Command binding.

#### Scenario: Code-behind is clean
- **WHEN** the code-behind file is inspected
- **THEN** no `OnRecordClick` method or `PointerPressed` event handler related to record selection SHALL exist
