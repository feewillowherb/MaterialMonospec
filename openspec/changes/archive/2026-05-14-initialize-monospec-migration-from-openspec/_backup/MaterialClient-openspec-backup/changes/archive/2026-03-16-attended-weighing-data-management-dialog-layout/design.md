## Context

本次变更旨在为 `AttendedWeighingWindow` 中的“数据管理”按钮新增一个台账列表对话框，用于在收料称重场景中快速浏览历史记录。该对话框仅用于 UI 布局与样式验收，参考 `Samples/Index.xaml` / `Index.xaml.cs` 以及提供的截图，实现类似的表格列、查询条件区和分页区布局，不涉及真实数据查询逻辑。现有 Avalonia 客户端已具备基础窗口框架与样式资源，需要在其上扩展一个新的对话框视图。

## Goals / Non-Goals

**Goals:**
- 在 `AttendedWeighingWindow` 中点击“数据管理”按钮时，弹出一个模态对话框，展示台账列表界面。
- 采用 Avalonia 控件绘制表头、数据行、查询条件控件、分页按钮等，与示例截图基本一致。
- 对话框内部使用一条硬编码测试数据行（或极少量测试数据）用于验收样式和列布局。
- 为后续接入真实数据和交互逻辑预留视图/视图模型结构，但本次不强制实现。

**Non-Goals:**
- 不实现真正的查询过滤、分页、导出等业务逻辑。
- 不接入后端 API 或数据库，不做数据持久化。
- 不修改现有称重业务流程或接口协议，只在 UI 层新增展示界面。

## Decisions

- **对话框触发方式**：在 `AttendedWeighingWindow.axaml` 中，将“数据管理”按钮绑定到一个命令或点击事件，弹出新的 Avalonia 对话框窗口（如 `Window` 或 `DialogHost`/自定义对话框控件），以保持与现有界面风格一致。
- **UI 结构划分**：对话框内采用垂直布局，上方为查询条件区域（文本框、日期选择器、下拉框、状态选择等），中间为 `DataGrid`/`DataGrid` 风格的表格控件，下方为分页和操作按钮区域（首页/上一页/下一页/末页/跳转页、确认/关闭等），布局参考 `Samples/Index.xaml`。
- **测试数据组织形式**：在对话框对应的 ViewModel 或代码隐藏中创建一个只读集合，包含一条代表性台账记录（含运单号、车牌号、类型、商品、状态、运单量、扣量、实际数量、实际重量、单位换算、进场时间等字段），页面加载时直接绑定该集合用于渲染。
- **样式与主题**：尽量复用项目现有样式资源（按钮样式、表头背景、行选中颜色等），在必要时针对该对话框定义局部样式，以确保与截图和既有 UI 一致，同时避免对全局样式产生破坏性影响。
- **可扩展性**：对话框的 ViewModel 预留属性和命令（查询条件、分页信息、确认操作等），以便后续在 /opsx:apply 后逐步填充真实业务逻辑，而无需大幅重构 UI。
- **Avalonia 表格与编译绑定**：表格使用 `DataGrid` 且以 `ItemsSource` 绑定集合（不用 WPF 的 `Items`），不设置 `CanUserAddRows`。在启用编译绑定的前提下，仅在对话框根节点设置 `x:DataType` 与对应 `xmlns`，不在 `DataGrid` 上设置会覆盖 DataContext 的 `x:DataType`，以保证 `ItemsSource` 与列绑定均能正确编译。

## Risks / Trade-offs

- 若测试数据字段命名或类型与未来后端实际模型不一致，后续接入真实数据时需要调整绑定字段；本次优先保证布局与样式，对模型命名保持尽可能贴近现有 Domain，但仍存在一定调整成本。
- 若对话框样式与已有窗口存在轻微差异（如字体、行间距），后续可能需要统一 UI 规范进行微调；为降低风险，本次尽可能复用现有样式资源。
- 由于本次不实现真实交互逻辑，部分按钮（查询、分页）将处于“无行为”或简单占位状态，可能需要在 UI 上通过禁用或说明文案避免用户误解为功能已可用。
