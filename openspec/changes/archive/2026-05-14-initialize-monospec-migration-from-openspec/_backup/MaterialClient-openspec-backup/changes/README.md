# OpenSpec 变更

本目录存放 MaterialClient 项目的变更提案。

## 目的

变更提案用于记录对系统的计划修改，包括新功能、缺陷修复、重构与流程改进。

## 目录结构

```
changes/
├── <change-id>/
│   ├── proposal.md     # 变更提案（必填）
│   ├── design.md       # 设计文档（可选）
│   └── tasks.md        # 实施任务
├── archive/            # 已完成的变更
└── _template/          # 变更提案模板
```

## 变更提案流程

1. **创建**：使用模板在 `<change-id>/` 目录下创建新提案
2. **设计**：如需技术设计则添加 design.md
3. **任务**：在 tasks.md 中拆解任务
4. **评审**：团队评审与批准
5. **实施**：按 tasks.md 执行任务
6. **归档**：完成后移入 `archive/`

## 状态分类

- **草稿**：初始提案，讨论中
- **已批准**：可开始实施
- **进行中**：实施中
- **已完成**：变更已实施并已移入归档

## 模板

使用 `_template/` 中的模板以保持格式一致。
